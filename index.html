Tabla con las diferencias entre Git Local y Git Remoto

Git Local Git Remoto
  • Git Local funciona en nuestra PC y sin conexion (en trabajo local)
  • Git Local es un sistema de control de versiones
  • Es adecuado para trabajar individualmente en un proyecto en tu PC
  • Puedes tener múltiples copias del repositorio local en diferentes carpetas de PC
  • Puedes crear y trabajar en ramas locales sin necesidad de conexión a Internet
  • Es útil para el desarrollo individual y experimentación
    		•
  • Git Remoto funciona en la nube 100%
  • Es la red social del codigo, una hoja de vida que demuestra tus habilidades
  • Facilita la colaboración entre múltiples desarrolladores
  • Normalmente, hay una única copia centralizada en el servidor remoto, a la que varios desarrolladores pueden acceder
  • Las ramas remotas rastrean las ramas del repositorio remoto
  • Facilita la colaboración, el seguimiento del progreso del proyecto y el respaldo del código en un lugar centralizado
